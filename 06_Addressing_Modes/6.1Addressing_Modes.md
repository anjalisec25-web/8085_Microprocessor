# Addressing Modes in 8085

## What are Addressing Modes?

Addressing modes specify how an instruction accesses its operand (data). They determine where the data is located and how the microprocessor retrieves it.

---

## Types of Addressing Modes

### 1. Immediate Addressing Mode

The data is provided directly in the instruction.

**Example:**
```assembly
MVI A, 25H
```

**Explanation:**
- `25H` is directly stored in register **A**.
- No memory location is accessed to fetch the data.

---

### 2. Register Addressing Mode

The operand is stored in a register.

**Example:**
```assembly
MOV A, B
```

**Explanation:**
- The contents of register **B** are copied to register **A**.

---

### 3. Direct Addressing Mode

The memory address of the operand is specified in the instruction.

**Example:**
```assembly
LDA 2500H
```

**Explanation:**
- The data stored at memory location **2500H** is loaded into the accumulator.

---

### 4. Register Indirect Addressing Mode

The memory address is stored in a register pair.

**Example:**
```assembly
MOV A, M
```

**Explanation:**
- The HL register pair contains the memory address.
- The data from that memory location is copied to register **A**.

---

### 5. Implicit Addressing Mode

The operand is implied by the instruction. No operand is specified explicitly.

**Example:**
```assembly
CMA
```

**Explanation:**
- The accumulator is complemented automatically.
- No register or memory address is mentioned in the instruction.

---

## Summary

| Addressing Mode | Operand Location | Example |
|-----------------|------------------|---------|
| Immediate | Data is part of the instruction | `MVI A, 25H` |
| Register | Register | `MOV A, B` |
| Direct | Memory address given in the instruction | `LDA 2500H` |
| Register Indirect | Memory address stored in a register pair | `MOV A, M` |
| Implicit | Operand is implied by the instruction | `CMA` |
