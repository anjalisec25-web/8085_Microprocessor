# Timing Diagrams in 8085

## What is a Timing Diagram?

A **Timing Diagram** is a graphical representation of the signals generated by the 8085 during the execution of an instruction or machine cycle. It shows the sequence and timing of various control signals.

---

## Common Timing Diagrams

| Timing Diagram | Description | T-States |
|----------------|-------------|:--------:|
| **Opcode Fetch** | Fetches the opcode from memory. It is the first machine cycle of every instruction. | **4–6** |
| **Memory Read** | Reads data from a specified memory location. | **3** |
| **Memory Write** | Writes data to a specified memory location. | **3** |
| **I/O Read** | Reads data from an input device. | **3** |
| **I/O Write** | Writes data to an output device. | **3** |
| **Interrupt Acknowledge** | Acknowledges an interrupt request from an external device. | **6** |

---

## Purpose of Timing Diagrams

- Understand the sequence of operations.
- Show the relationship between address, data, and control signals.
- Help in designing and debugging interfacing circuits.
