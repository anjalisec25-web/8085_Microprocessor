# Machine Cycle and T-State

## What is a T-State?

A **T-State (Timing State)** is the time required to complete one clock pulse of the microprocessor. It is the smallest unit of time in 8085 operations.

---

## What is a Machine Cycle?

A **Machine Cycle** is the time taken by the microprocessor to complete one basic operation, such as reading from memory or writing to memory. A machine cycle consists of one or more T-States.

---

## Types of Machine Cycles

### 1. Opcode Fetch Cycle
- Fetches the instruction (opcode) from memory.
- It is the first machine cycle of every instruction.
- Usually takes **4 to 6 T-States**.

### 2. Memory Read Cycle
- Reads data from a memory location.
- Usually takes **3 T-States**.

### 3. Memory Write Cycle
- Writes data to a memory location.
- Usually takes **3 T-States**.

### 4. I/O Read Cycle
- Reads data from an input device.
- Usually takes **3 T-States**.

### 5. I/O Write Cycle
- Writes data to an output device.
- Usually takes **3 T-States**.

### 6. Interrupt Acknowledge Cycle
- Acknowledges an interrupt request from an external device.

---

## Difference Between T-State and Machine Cycle

| T-State | Machine Cycle |
|----------|---------------|
| Smallest unit of time. | Group of one or more T-States. |
| Equal to one clock pulse. | Performs one basic operation. |
| Measured in clock cycles. | Measured in T-States. |
