# Interrupts in 8085

## What is an Interrupt?

An **Interrupt** is a signal that temporarily stops the normal execution of a program and transfers control to a special service routine. After the interrupt is serviced, the processor resumes execution of the main program.

---

## Types of Interrupts

### 1. Hardware Interrupts
Hardware interrupts are generated by external devices connected to the microprocessor.

**Hardware Interrupts in 8085:**
- TRAP
- RST 7.5
- RST 6.5
- RST 5.5
- INTR

---

### 2. Software Interrupts

Software interrupts are generated by executing interrupt instructions in a program.

**Software Interrupts:**
- RST 0
- RST 1
- RST 2
- RST 3
- RST 4
- RST 5
- RST 6
- RST 7

---

## Interrupt Priority

| Priority | Interrupt |
|:--------:|-----------|
| 1 (Highest) | TRAP |
| 2 | RST 7.5 |
| 3 | RST 6.5 |
| 4 | RST 5.5 |
| 5 (Lowest) | INTR |

---

## Features of TRAP

- Highest priority interrupt.
- Non-maskable interrupt.
- Edge and level triggered.

---

## Features of INTR

- Lowest priority interrupt.
- Maskable interrupt.
- Requires an interrupt acknowledge signal (INTA).

---

## Note

The instructions **EI**, **DI**, **SIM**, and **RIM** are used for interrupt control and are discussed in the **Instruction Set → Control** section.
